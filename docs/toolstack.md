# ツールスタック

## 方針

- 汎用部品はライブラリを活用する
- フレームワーク自体の依存は最小限に保つ
- 開発時のみ必要なツールは「開発環境」カテゴリに分離
- 型安全性を重視し、静的型チェック・ランタイムバリデーション・データバリデーションの3層で堅牢性を確保

## バリデーション3層

| 層                           | 何を検証するか                   | ツール       |
| ---------------------------- | -------------------------------- | ------------ |
| **静的型チェック**           | 関数シグネチャ・変数の型整合性   | **pyright**  |
| **ランタイムバリデーション** | 設定ファイル（YAML等）の構造・値 | **Pydantic** |
| **データバリデーション**     | DataFrameのカラム型・値制約      | **Pandera**  |

## フレームワーク実行に必須

フレームワーク利用者にも要求されるツール。

| ツール           | 用途                                                               |
| ---------------- | ------------------------------------------------------------------ |
| **Python 3.11+** | 実行環境                                                           |
| **uv**           | パッケージ管理・仮想環境・Pythonバージョン管理                     |
| **DuckDB**       | インメモリクエリエンジン（`:memory:` モード、永続DBなし）          |
| **Polars**       | DataFrame操作（イミュータブル・Arrow形式でDuckDBとゼロコピー連携） |
| **Pydantic**     | 設定・スキーマの型定義とランタイムバリデーション                   |
| **Pandera**      | DataFrameバリデーション（カラム型・値制約の検証）                  |
| **networkx**     | DAG構造の構築・走査・検証                                          |
| **DVC**          | データバージョン管理・パイプライン実行                             |
| **ruamel.yaml**  | YAML 1.2パーサー（コメント・書式のラウンドトリップ保存）           |
| **Typer**        | CLIフレームワーク（型ヒントベース、Click上に構築）                 |
| **Git**          | ソース・設定・メタデータのバージョン管理                           |

## 開発環境

フレームワーク自体の開発・CI・品質維持に使用。利用者にも推奨。

### リンター・フォーマッター

| ツール       | 対象                 | 用途                               |
| ------------ | -------------------- | ---------------------------------- |
| **Ruff**     | Python               | リンター + フォーマッター          |
| **Prettier** | Markdown, YAML, JSON | 非Pythonファイルのフォーマット統一 |

### 型チェッカー

| ツール      | 用途                                                    |
| ----------- | ------------------------------------------------------- |
| **pyright** | 静的型チェック（CI + VS Code Pylance バックエンド共通） |

### テスト・CI

| ツール             | 用途                                                |
| ------------------ | --------------------------------------------------- |
| **pytest**         | テストフレームワーク                                |
| **pytest-cov**     | カバレッジ計測                                      |
| **pre-commit**     | コミット時の自動チェック（Ruff, Prettier, pyright） |
| **GitHub Actions** | CI（lint → type-check → test）                      |

### エディタ・AIサービス

| ツール                        | 用途                                |
| ----------------------------- | ----------------------------------- |
| **VS Code**                   | エディタ                            |
| **Claude Code**               | ターミナルベースAIコーディング      |
| **GitHub Copilot**（VS Code） | エディタ内AIアシスト                |
| **gh**                        | GitHub CLI（Issue駆動開発・PR管理） |

## 参考（フレームワーク責務外）

フレームワーク設計には影響しないが、主要利用者の環境として把握しておくもの。

| ツール            | 用途                 | 備考                                                |
| ----------------- | -------------------- | --------------------------------------------------- |
| **marimo**        | リアクティブnotebook | .pyファイルのためGit差分がクリーン                  |
| **pandas**        | DataFrame操作        | AP層でユーザーが使用する場合、`.to_pandas()` で変換 |
| **numpy / scipy** | 科学計算             | ユーザーが必要に応じて導入                          |
